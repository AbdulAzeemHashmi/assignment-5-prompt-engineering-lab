# Assignment #5: Prompt Engineering Battle Lab

This repository contains the deliverables for the Prompt Engineering Battle Lab. It demonstrates the systematic application of prompt engineering techniques for software development, debugging, refactoring, and code reviewing, while strictly maintaining the Git workflow.

## Objectives
- Compare bad, better, and professional prompts to understand structure and efficacy.
- Write constraint-based prompts for code generation.
- Leverage AI for debugging with context-rich prompts.
- Refactor messy code for readability and maintainability.
- Perform strict AI code reviews and apply feedback.
- Document token usage and responsible AI practices.

## Technologies and Tools
- Language: JavaScript (Node.js)
- Development Environment: VS Code
- Git & GitHub for issue tracking, pull requests, and code versioning
- AI tools: Gemini (Antigravity AI Coding Assistant)

## Prompting Techniques Used
- **Role Play**: Assigning a specific persona (e.g., Senior JavaScript Mentor, Strict Code Reviewer).
- **Context Injection**: Providing expected inputs, outputs, errors, and system context.
- **Constraints**: Setting clear rules (e.g., "no external libraries", "no DOM manipulation", "reusable code").
- **Output Formatting**: Requesting structured outputs (e.g., code first, then explanation).
