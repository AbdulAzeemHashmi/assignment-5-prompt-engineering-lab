# Responsible AI Usage Policy

This policy governs the responsible use of Generative AI tools (such as Gemini, Claude, ChatGPT, Cursor, and GitHub Copilot) during software development. It outlines code safety rules, data privacy rules, verification processes, and transparency standards.

## 1. Data Privacy and Security Constraints

To prevent data leaks, developers **must never** input the following sensitive information into public or unverified AI models:
- **API Keys & Credentials**: Do not share live connection strings, private certificates, or access tokens.
- **Environment Variables (.env)**: Never paste configuration files that define local or production environments.
- **Personal Identifiable Information (PII)**: Do not include emails, phone numbers, real names, or addresses of users/clients in prompts.
- **Company IP**: Avoid pasting proprietary business logic or codebase chunks that are confidential.

## 2. Output Verification and Testing

Developers are solely responsible for the correctness, performance, and security of any code generated by AI.
- **Review every line**: Do not "blindly copy" code blocks. Walk through the logic line-by-line to understand its flow.
- **Manual and Automated Testing**: Ensure that all generated logic is covered by unit tests or manual test runs.
- **Security Check**: Check the code for potential vulnerabilities (e.g., SQL injection, unsafe regex, or deprecated functions).

## 3. Honest AI Disclosure

To build trust and document code origins:
- **Keep Prompts and Answers Documented**: Maintain a record of significant prompts, AI outputs, and manual corrections.
- **Code Provenance**: Clearly distinguish between purely custom logic and AI-assisted generation in commits.
- **Limitations Recognition**: Acknowledge when AI is helpful (e.g., boilerplates, regex generation) versus where manual intervention is required (critical business logic, security).
